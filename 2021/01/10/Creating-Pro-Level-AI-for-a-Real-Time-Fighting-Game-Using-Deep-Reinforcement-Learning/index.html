Creating Pro-Level AI for a Real-Time Fighting Game Using Deep Reinforcement Learning

Posted on 2021-01-10 Edited on 2021-06-16 In 强化学习

Creating Pro-Level AI for a Real-Time Fighting Game Using Deep Reinforcement Learning

摘要

​ 强化学习与深层神经网络相结合最近在许多类型的游戏中表现得非常好。它在固定的游戏环境和基于回合制的两个玩家棋盘游戏已经超过了人类水平的表现。然而，据我们所知，目前的研究还没有在现代复杂的战斗游戏中超过人类水平的表现。这是由于实时战斗游戏固有的困难，包括巨大的动作空间、动作依赖和不完善的信息。 我们克服了这些挑战，并为商业游戏“Blade & Soul”制作了1v1战斗AI代理”。 经过训练的代理与五名职业玩家竞争，取得了62%的胜率。

​ 本文提出了一种实用的强化学习方法，包括一种新的self-play curriculum和data skipping技术。通过curriculum，通过奖励塑造创造了三种不同风格的代理，并相互训练。 此外本文还提出了data skipping技术，可以提高数据效率，促进在巨大空间探索。

介绍

​ 强化学习(RL)正在将其边界扩展到各种游戏类型。在PVE（玩家与环境）环境中，例如在Atari2600游戏中发现的环境，使用各种方法，RL代理已经超过了人类水平的性能。 同样在PVP（玩家对玩家）环境中，神经网络与基于搜索的方法相结合在两个玩家的回合游戏中击败了最好的人类玩家，比如围棋和国际象棋。最近，游戏中的RL研究已经转移到更复杂的PVP电子游戏中，例如StarCraft2 ，Quake3 和Dota2。

​ 战斗游戏作为最具代表性的复杂PVP游戏之一，一直是在这一领域取得进展的多项研究的重点。例如，基于MCTS的方法已应用于游戏平台“FightingICE”,这是一个为战斗游戏AI竞争而制作的游戏平台。然而当应用于巨大的现代游戏引擎时，查询时间较长，很难满足实时条件。此外，在Little Fighter 2(LF2)中一种深度强化学习基础代理通过一个基于规则的固定对手训练。 然而由于对手的决定是未知的，通过基于规则训练的AI不能泛化到看不见的对手。我们的方法在很大程度上类似于self-play RL方法被应用在“Super Smash Bros. Melee (SSBM)”。然而与我们具有复杂游戏规则的3D环境相比，状态和动作空间的复杂性受到了显著的限制。 我们通过新颖的基于self-play RL，为实时战斗游戏““Blade & Soul (B&S) Arena Battle”创建了专业水平的人工智能代理。

​ B&S是一款商业化的大型多人在线角色扮演游戏，它支持两名玩家之间的决斗，称为“B&S Arena Battles (BABs)”。如表1所示，与其他作品中考虑的游戏相比，BAB是一种更现代的战斗游戏；因此，它具有更复杂的游戏动力学和更大的游戏引擎。此外，大量的人玩BAB和它有更活跃的专业场景比其他战斗游戏。与专门为研究目的而设计的FightingICE相比，BAB有更多的活跃专业场景。

​ 图1显示了一个来自BAB的场景。 BAB是一个两人零和游戏。 在BAB中，两名玩家互相对抗，以在三分钟内将对手的HP（血量）降低到零。 要掌握BAB，代理必须能够应对多种挑战。

​ 首先，代理必须管理巨大的动作和状态空间。代理必须同时进行技能、移动和目标决策，从而产生许多可能的组合。 粗略估计，每个时间步有144个潜在动作：8(avg. # of avail. skills)9（8 directional + no move)2(facing opponent or moving direction）。 由于平均游戏长度为1200个时间步长(120s)，许多场景是可能的,而不是考虑对手的动作。

​ 此外，代理必须考虑技能之间的依赖关系：例如在使用另一种技能之后，一种技能只能在短时间内可用。 因此，在总共45种技能中（including “no-op”），技能随着时间不断的变化。 代理还必须考虑每种技能的特性，因为它们具有不同的冷却时间（重用技能所需的间隔）和SP（技能点）消耗，并服务于五种不同的功能中的一种或多种： 攻击，群体控制(功能使对手无能力；缩写CC)，抵抗(功能使玩家免疫或抵抗CC技能)，逃跑和冲刺。

​ 最后，代理必须处理不完善的信息设置。 因为BAB是一个实时游戏，两个玩家同时做出决定。 这表明代理人必须在不知道对手的决定或策略的情况下作出决定。 因此，BAB可以被认为是一系列石头剪刀游戏。例如当玩家使用抵抗技能，而对手同时使用群体控制技能时，玩家比对手获得优势。 因此，问题的本质是逼近纳什均衡策略，使代理能够对任何相反的策略作出适当的反应。为了应对这些挑战，我们通过多样化对手池对普通的self-play算法进行了改进和通过数据跳过技术来促进探索。 这项工作的主要贡献如下：

  • 我们设计了一个新的self-play 课程和不同风格的代理。 课程使这些代理人相互竞争，同时加强代理人，使代理人能够处理各种对手。 我们的经验证明，我们的课程优于普通的自我游戏方法。
  • 我们通过奖励塑造)来多样化游戏AI的战斗风格。 我们创造了三种类型的代理，具有不同的战斗风格：好斗型、防御型、平衡型。 我们预计它将应用于游戏开发，包括水平设计和自动平衡。
  • 我们引入了数据跳过技术来增强在广阔空间中的探索。 这些通常可以应用于任何两人实时战斗游戏。
  • 我们通过在2018年B&S World Championship Blind Match中与专业球员对位来评估我们的代理。 我们的人工智能代理赢得了七场比赛中的三场，而其中一个好斗的代理在现场比赛和预试中击败了所有专业玩家。

多风格Self-Play Curriculum

​ 现有的自我游戏方法 通常使用对手池进行训练。 在训练期间，网络的参数定期存储，以创建一个过去的自我池。 然后从这个池中取样对手。

​ 虽然RL的自我游戏方法提供了一种学习纳什均衡策略的方法，但策略空间的高覆盖率对于有效地找到策略空间是必不可少的。 普通的自我游戏本身并不能保证足够的覆盖范围对于有大问题空间的游戏。 为了解决这个问题，Alphastar通过模仿不同的人类策略，引入三种不同匹配方案的代理，使对手池多样化。 扑克AI，Pluribus在基本蓝图策略之上手工调整了三种不同的策略。 这三种策略分别偏向于加注、跟进和弃牌。

​ 同时，我们设计了一种新颖的自我游戏课程。 我们通过引入一系列不同的战斗风格来加强代理策略的多样性，并使不同风格的代理相互竞争。

通过奖励塑造指导战斗风格

​ 最值得注意的战斗风格之一是侵略性的程度。 我们使用奖励的三个维度来控制攻击性的程度。 第一个维度是“时间惩罚”。 侵略性的代理每一步都会受到更大的惩罚，这促使它在更短的时间内完成比赛。 第二个维度是代理的HP对对手的HP的相对重要性。 侵略性的代理会试图减少对手的HP，而不是保持自己的HP，而防守w玩家往往采取相反的方式。 最后一个维度是“距离惩罚”。 防守型玩家倾向于确保与对手的一定距离，以适当地应对攻击，而攻击性玩家则倾向于接近对手并无情地攻击。 为了实现这些特性，与自身与其对手之间的距离相比，攻击性代理受到了更大的惩罚。每个样式使用的具体奖励权重如表2所示。 请注意，这三个维度中的每一个都可以取连续值。 这意味着有可能创建一系列不同的战斗风格。然而，为了有效地证明这种方法的可行性，我们限制了这种方法战斗风格的数量为三种。 通过使用任何类型的附加奖励信号以及和，该方法可以应用于一般的其他战斗游戏，以创建具有不同战斗风格的代理。

Our Self-Play Curriculum

​ 图3显示了三种不同类型代理的自我游戏课程的概述。 每种类型的代理都有自己的学习过程，所有三种代理类型都是以并发的方式进行训练。

​ 每个学习过程由一个学习者和多个模拟器组成。 学习者和模拟器异步工作。 在模拟器中，代理不断地与来自共享池的随机抽样对手进行对弈。 每种样式的最近k个模型被均匀地选择，总概率质量为p，而其他模型被均匀地选择概率1-p。随着训练的进行，p从0.8到0.1线性退火。 较高的p有助于快速适应最新的对手，而较低的p通过减轻灾难性遗忘来稳定学习过程。 每个模拟器在每次匹配结束时向学习者发送匹配日志，并使用从学习者收到的最新参数更新其代理。 同样的程序继续通过后续游戏使用。

​ 学习者使用从多个模拟器收集的日志以off-policy的方式训练其代理，并根据请求将最新的网络参数发送给模拟器。 此外，学习者将其网络参数发送到共享池的每一个C步(例如C=10,000)更新。 因此，池子具有不同的策略，这些策略来自不同风格的不同学习过程。 这些模型参数作为每个学习过程的对手提供。通过共享一个池，每个学习代理在训练中遇到每种风格的对手，并学习如何处理他们。 因此，通过我们的自我游戏课程培训的代理最终可以学习如何面对不同战斗风格的对手，同时保持自己的战斗风格。

数据跳过技术

​ 在本节中，我们详细介绍了数据跳过技术，它是指在训练和评估过程中丢弃某些数据的过程。

丢弃没合法动作的数据

​ “no-op”表示不做任何行动。

​ 在战斗游戏中，使用技能通常消耗资源，如SP和冷却时间。 因此，如果玩家过度使用某一技能，它将无法在实际需要的时候使用。 因此，玩家应该战略性地使用和保留他们的技能，以确保他们在需要时的可用性。 为了考虑到这一点，我们将“no-op”操作连接到策略网络的输出，允许代理选择“no-op”，并在必要时在一定时期内不做任何事情。 这意味着我们的行动空间有44种技能，加上额外的“no-op”行动。 这一点很重要，因为BAB的人类游戏日志表明，“no-op”动作占人类玩家技能使用的最大比例。

​ “no-op”决策可分为被动使用和主动使用。 被动使用“no-op”意味着代理选择“no-op”，因为没有可用的技能。 例如，当代理耗尽资源或被对手的CC技能击中时，代理别无选择，只能选择“no-op”。 积极使用“no-op”意味着代理从战略上选择“no-op”，即使其他技能可以使用。

​ 我们从训练和评估阶段丢弃了被动的“no-op”数据，因为被动的“no-op”不是由代理故意使用的。 此外，该方法使LSTM能够反映较长时间层的表示，因为数据没有提供给网络。 我们在实验部分表明，跳过被动“no-op”大大提高了学习效率。 请注意，这种方法通常适用于其他领域，其中可用的技能不断变化，“no-op”操作是一个有效的选择。

保持移动行动

​ 虽然单个技能决策可以对后续状态产生实质性影响，但单个移动决策的效果相对有限。 原因是一个角色在一个时间步长(0.1s)中移动的距离很短，考虑到它的速度。 为了使任何移动决策产生有意义的效果，代理应该做出相同连续移动的决定。 这允许代理从字面上“move”，并导致后续状态和奖励的变化。 因此，很难从初始策略中训练具有随机移动决策的移动策略。 由于一项随机政策连续作出相同决定的机会很低，探索是极其有限的。 因此我们建议维持固定时间步骤的移动决定。

​ 图4显示了该方法如何与示例一起工作。 如果代理选择移动操作，则会跳过以下n-1时间步骤的移动决策。 这意味着代理总共对n个步骤保持相同的移动决策。 请注意，我们的方法与帧跳过技术有不同的目的(在Atari领域。 为了提高模拟器的效率，引入了跳帧技术。 然而，我们不能仅仅跳过框架，因为技能决策仍然必须进行。 虽然我们不能享受模拟器的效率优势，但保持移动仍然有利于训练，这仅仅是因为保持移动决策会增加单个移动决策的影响，正如我们将通过实验证实的那样。 从这个意义上说，保持移动可以被看作是“放大优势”。

​ 根据表4中的数据，在侵略性代理的情况下，使用共享池的效果是很小的。 它表明，无论是否提供不同的对手，进行训练的策略空间是相似的。 这与战斗游戏的性质有关，如果另一方接近并发起一场战斗，其中一方应该反击。 因此，在一个攻击性的代理人的情况下，攻击始终如一，有一点不同的经验，无论对手的战斗风格的多样性。

丢弃没合法动作的数据的效果

​ 正如上一节所讨论的那样，“no-op”决定可以是主动的，也可以是被动的。 我们进行了一个实验，以研究从学习中丢弃这种被动的“no-op”数据的效果。 实验的训练伙伴是内置的BAB AI，其水平与前20%的玩家相当。 我们测量了代理学会击败它的速度，结果如图5(a)所示)。 如果从学习数据中丢弃“no-op”，则在70k步后，获胜率达到80%，而当包含“no-op”数据时，则需要170k步。 当跳过被动的“no-op”数据时，达到90%赢率所需的时间步长减少到一半。本实验证实，通过从学习数据中丢弃被动“no-op”，提高了训练性能。

玩家评估

​ 本节将讨论预赛和盲赛的结果，以及确保人类选手公平的条件。

公平的条件

反应时间

​ 当人类在实时战斗游戏中面对人工智能时，影响结果的最重要因素是反应时间。 人类需要一些时间来识别对手使用的技能，并通过移动他的手按下按钮。一个人工智能的决定要反映在游戏中我们应用平均230毫秒的延迟，这样人工智能就没有优势了。 这一延迟量对应于BAB中职业玩家的平均反应时间。

种类和技能

​ 在B&S中有11个种类，每个种类都有独特的特点。 由于类之间存在相对优势，我们将AI和职业玩家的类固定为“Destroyer”。 “Destroyer”是一个有内讧风格的种类，并稳步出现在B&S世界锦标赛。 此外，AI和职业玩家的技能被设置为相同的，以确保公平的匹配。 根据BAB用户统计数据，选择大多数用户所选择的技能。

总结

​ 利用深度强化学习，我们创建了AI代理，在3D实时战斗游戏中与专业玩家战斗。 为了实现这一点，我们提出了一个用奖励塑造来指导战斗风格的方法。 采用三种代理风格，引入了一种新的自我游戏课程，以提高泛化性能。 我们还提出了数据跳过技术，以提高数据效率，并使有效的探索。 因此，我们的代理能够与世界上最好的BAB玩家竞争。 提出的训练方法一般适用于其他格斗游戏。

阅读总结

  • 本文提出了一种实用的强化学习方法，包括一种新的self-play curriculum和data skipping技术。通过curriculum，通过奖励塑造创造了三种不同风格的代理，并相互训练。
  • 通过reward shaping来创建不同风格的代理。
  • 在self-play过程中，同时训练不同风格的代理，并指定训练步数更新参数到agent pool中，用作对手训练。
  • 通过数据跳过技术来促进探索。
  • 跳过合法动作数量为0的数据，即被控制不能放出技能的数据等。通过实验证明能加速训练过程，达到90%的训练步数减少到一半。
  • 使用移动的相关动作时，同时连续做几个相同的移动动作，因为当个移动距离很短。
  • 网络结构输出mask掉不合法的动作
